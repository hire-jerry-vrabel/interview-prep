## Study Approach

**Topics** — I work through each technology area systematically,
writing notes, code examples, and gotchas I've actually encountered
in production. Not copy-pasted docs — real hard-won knowledge.

**Formats** — Each interview format has its own muscle memory.
I treat each one as a separate skill to practice deliberately,
not just something you wing on the day.

**Consistency over intensity** — A little every day beats a cram
session the night before. This repo is my daily log of that habit.
